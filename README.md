# アーティスト公式サイト DB設計

**制作者：** キム・ソンス（Kim Sungsu）

## 使用技術
- MySQL / JSP / Servlet / Java
- JavaMail API（Gmail SMTP）／ JSch（SFTP）／ SHA-256（パスワード暗号化）
- Apache Commons DBCP2（コネクションプール）／ Servlet Multipart API（ファイルアップロード）

---

本プロジェクトは、歌手「コ・ギョンビン（고경빈）」の公式ウェブサイトを想定して設計したデータベースです。  
アルバム、楽曲、動画、ギャラリー、公演情報、SNSリンク、お問い合わせ機能など、  
アーティスト活動に必要なコンテンツを統合的に管理できるように構成しました。

- リポジトリ：https://github.com/todtnwater/kokyoungbin
- サイトリンク : https://egocci.co.kr/egocci/Index

---

## 設計の目的

- アーティスト情報を体系的に管理する
- アルバムと楽曲を中心に音楽コンテンツを整理する
- 動画、ギャラリー、公演情報、SNSなどを一つのサイトで提供する
- 管理者がコンテンツを追加・更新しやすい構造にする

---

## 主要テーブル構成

### member（会員・管理者）
サイト利用者および管理者情報を管理するテーブルです。  
メールアドレス、パスワード、氏名、連絡先、権限情報などを保持します。

### albums（アルバム）
アルバム情報を管理するテーブルです。  
アルバム名、種類、発売日、説明、カバー画像などを保存します。

### songs（楽曲）
楽曲情報を管理する中心テーブルです。  
各楽曲はアルバムに属し、曲名、アーティスト名、ジャンル、歌詞、再生時間、タイトル曲などを管理します。

### streaming_platforms（配信プラットフォーム）
音源配信プラットフォーム情報を管理します。  
例：YouTube Music、Spotify、Melon などの外部配信サービスを想定しています。

### song_streaming_links（配信リンク）
楽曲ごとの配信リンクを管理します。  
1つの楽曲を複数のプラットフォームに接続できるように設計しました。

### videos（動画）
Music VideoやライブMV映像など、動画コンテンツを管理するテーブルです。

### gallery（ギャラリー）
アーティスト写真や活動画像を管理するテーブルです。

### scheduled_performances（公演情報）
公演やイベント日程を管理するテーブルです。  
公演名、会場、日付、時間、チケットリンク、ポスター画像などを保存します。

### social_links（SNSリンク）
SNSおよび外部リンクを管理するテーブルです。  
Instagram、YouTube、TikTok などへの導線を提供します。

### business_mail（お問い合わせ）
ビジネスお問い合わせ情報を管理するテーブルです。  
送信者情報、件名、本文、送信状態などを保存します。

---

## テーブル関係

- albums 1 : N songs
- songs 1 : N song_streaming_links
- streaming_platforms 1 : N song_streaming_links

この構造により、1つのアルバムに複数の楽曲を登録でき、  
さらに1つの楽曲を複数の音源配信サービスに連携できるようにしました。

---

## 設計の特徴

- 音楽コンテンツを中心に構造化した設計
- 楽曲と配信リンクを分離し、拡張しやすい形にした
- 公演、SNS、動画、ギャラリーなどを独立管理して保守性を向上
- 管理画面から各コンテンツを柔軟に追加・修正できることを想定

---

## 補足

本データベースは、単純な紹介ページではなく、  
アーティストの音楽活動・広報・ファン向け情報提供を一元的に管理できる  
公式サイトを想定して設計しました。
